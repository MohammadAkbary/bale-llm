## 🔤 دستورات عمومی

### `/start`

- **توضیح**: شروع مجدد ربات
- **دسترسی**: عمومی
- **پاسخ**: پیام خوش‌آمد

**مثال:**

```
کاربر: /start
ربات: سلام! من ربات هوشمند بله هستم 🤖
```

### `/help`

- **توضیح**: نمایش لیست دستورات و راهنما
- **دسترسی**: عمومی
- **پاسخ**: راهنمای کامل

**مثال:**

```
کاربر: /help
ربات: [نمایش لیست تمام دستورات]
```

### `/lang [کد‌زبان]`

- **توضیح**: تغییر زبان واسط
- **دسترسی**: عمومی
- **پارامترها**:
  - `fa` - فارسی
  - `en` - انگلیسی
- **پاسخ**: تایید تغییر زبان

**مثال:**

```
کاربر: /lang en
ربات: ✅ Language changed to English
```

## 👨‍💼 دستورات مدیری

### `/admin_list`

- **توضیح**: نمایش لیست مدیران فعلی
- **دسترسی**: فقط مدیران
- **پاسخ**: لیست Admin IDs

**مثال:**

```
کاربر (Admin): /admin_list
ربات: 📋 لیست مدیران:
     • 123456789
     • 987654321
```

### `/add_admin [USER_ID]`

- **توضیح**: اضافه کردن Admin جدید (فقط Admin موجود می‌تواند)
- **دسترسی**: فقط مدیران
- **پارامترها**:
  - `USER_ID` - شناسه‌ی کاربری (عددی)
- **پاسخ**: تایید اضافه کردن

**مثال:**

```
کاربر (Admin): /add_admin 777888999
ربات: ✅ مدیر جدید اضافه شد: 777888999
```

### `/add_user [USER_ID]`

- **توضیح**: اضافه کردن کاربر مجاز
- **دسترسی**: فقط مدیران
- **پارامترها**:
  - `USER_ID` - شناسه‌ی کاربری (عددی)
- **پاسخ**: تایید اضافه کردن

**مثال:**

```
کاربر (Admin): /add_user 111222333
ربات: ✅ کاربر جدید اضافه شد: 111222333
```

### `/remove_user [USER_ID]`

- **توضیح**: حذف کاربر مجاز
- **دسترسی**: فقط مدیران
- **پارامترها**:
  - `USER_ID` - شناسه‌ی کاربری (عددی)
- **پاسخ**: تایید حذف

**مثال:**

```
کاربر (Admin): /remove_user 111222333
ربات: ✅ کاربر حذف شد: 111222333
```

### `/user_list`

- **توضیح**: نمایش لیست کاربران مجاز
- **دسترسی**: فقط مدیران
- **پاسخ**: لیست User IDs

**مثال:**

```
کاربر (Admin): /user_list
ربات: 📋 لیست کاربران مجاز:
     • 111222333
     • 444555666
     • 777888999
```

## 📨 درخواست‌های معمولی

### پیش‌فرض: هر پیام دیگری

- **توضیح**: پیام‌های دیگری به AI ارسال می‌شوند
- **دسترسی**: فقط کاربران مجاز و مدیران
- **پاسخ**: جواب AI

**مثال:**

```
کاربر: سلام، هوا امروز چطور است؟
ربات: [جواب هوشمند از Google Gemini]
```

## ❌ پیام‌های خطا

### `❌ شما دسترسی به این عملیات ندارید`

- **دلیل**: کاربر سعی دستور Admin را اجرا کرد
- **راه‌حل**: فقط مدیران می‌توانند این را انجام دهند

### `❌ کاربر یافت نشد`

- **دلیل**: User ID اشتباه یا وجود ندارد
- **راه‌حل**: User ID را بررسی کنید

### `⚠️ این کاربر قبلاً اضافه شده`

- **دلیل**: کاربر/مدیر قبلاً موجود است
- **راه‌حل**: نیازی نیست دوباره اضافه کنید

### `❌ شناسه کاربر نامعتبر`

- **دلیل**: User ID عددی نیست
- **راه‌حل**: فقط اعداد استفاده کنید

### `❌ شما دسترسی به این ربات ندارید. لطفاً با مدیران تماس بگیرید`

- **دلیل**: کاربر در لیست مجاز نیست
- **راه‌حل**: Admin باید شما را اضافه کند

## ⚙️ تنظیمات قابل کنترل

### `require_approval`

- **پیش‌فرض**: `true` (کاربران نیاز به تأیید دارند)
- **مقادیر**: `true` یا `false`
- **توضیح**: اگر `true` باشد، فقط کاربران مجاز و مدیران می‌توانند از ربات استفاده کنند

**تغییر در کد:**

```python
db.set_setting('require_approval', False)  # اجازه همه
```

## 📊 مثال جریان کامل

```
1️⃣ کاربر جدید پیام ارسال می‌کند
   ↓
2️⃣ سیستم بررسی می‌کند: آیا مجاز است؟
   ↓
3️⃣ اگر مجاز نبود: ❌ پیام رد شد
   ↓
4️⃣ اگر مجاز بود: پیام به AI ارسال می‌شود
   ↓
5️⃣ AI جواب می‌دهد
   ↓
6️⃣ پاسخ به کاربر ارسال می‌شود
```

## 💾 ذخیره‌سازی داده‌ها

تمام داده‌ها در فایل `bot_database.json` ذخیره می‌شوند:

```json
{
  "admins": ["123456789", "987654321"],
  "authorized_users": ["111222333", "444555666"],
  "blocked_users": [],
  "settings": {
    "require_approval": true,
    "auto_add_users": false
  }
}
```

## 🐛 debugging

برای debugging، فایل `main.py` را با `FLASK_DEBUG=True` اجرا کنید:

```bash
FLASK_DEBUG=True python main.py
```

لاگ‌های تفصیلی در console نمایش داده خواهند شد.
